## Best Practices for Context7 Integration

### 1. **Progressive Enhancement**
Start with basic functionality and gradually add features:

```
1. Basic CRUD operations
2. Authentication and authorization
3. Advanced features (search, pagination, etc.)
4. Performance optimization
5. Monitoring and logging
```

### 2. **Version-Specific Requests**
Always specify versions for critical dependencies:

```
✅ "Next.js 14 App Router with React 18"
✅ "FastAPI with Python 3.11+ and SQLAlchemy 2.0"
✅ "React Native with Expo SDK 50"
```

### 3. **Complete Stack Integration**
Request full stack solutions rather than individual components:

```
✅ "Full-stack e-commerce with Next.js, Prisma, Stripe, and Vercel"
✅ "Mobile app with React Native, Firebase, and app store deployment"
✅ "Microservices with Docker, Kubernetes, and monitoring"
```

### 4. **Production-Ready Code**
Always request production considerations:

```
✅ "Include error handling, logging, monitoring, and deployment config"
✅ "Add proper TypeScript types and validation"
✅ "Include testing setup and CI/CD pipeline"
```

## Troubleshooting Context7 Requests

### Common Issues and Solutions:

1. **No Results for Specific Library**
   - Try alternative names or broader terms
   - Check if the library is supported on context7.com
   - Use related libraries as examples

2. **Outdated Information**
   - Specify version numbers in your requests
   - Ask for "latest" or "current" patterns
   - Request specific features or APIs

3. **Incomplete Examples**
   - Be more specific about what you need
   - Request "complete" or "full" implementations
   - Ask for "production-ready" code

4. **Integration Issues**
   - Request "integration" or "setup" examples
   - Ask for "configuration" files
   - Request "step-by-step" instructions
